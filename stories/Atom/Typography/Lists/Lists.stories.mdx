# Lists
A List consists of related items grouped together.

### Overview
Lists are used to display data in a linear way. When a bunch of information is to be conveyed, it can be grouped together in a list and can be mentioned in a systematic way so that the readers find it more effective to read.

#### When to use:
- To arrange related information in a systematic way.
- To describe information in a hierarchical structure.

### Formatting

#### Default
The List consists of information arranged one below another. It can be in a hierarchical or descriptive format. 

### Content

The list consists of three types: Unordered, Ordered and Description.

1. **Unordered List**: An unordered list is used if there is no hierarchy required. They start with a bullet. For starting an unordered list, ul tag is used. A li tag is added for each list item.

<Canvas>
  <Story name="Unordered List">
    {(args, { globals: { locale } }) => {
      const caption = getCaptionForLocale(locale);
      return <List data={caption} type="ul"></List>;
    }}
  </Story>
</Canvas>

2. **Ordered List**: An ordered list is used if there is a relation/hierarchy between list items. They generally begin with numbers. For starting an ordered list, an ol tag is used. A li tag is added for each list item.

<Canvas>
  <Story name="Ordered List">
    {(args, { globals: { locale } }) => {
      const caption = getCaptionForLocale(locale);
      return <List data={caption} type="ol"></List>;
    }}
  </Story>
</Canvas>

3. **Description List:** A description list is a list of terms, with a description of each term. The d1 tag defines the term (name), and the dd tag describes each term.

<Canvas>
  <Story name="Description List">
    {(args, { globals: { locale } }) => {
      const caption = getCaptionForLocale(locale);
      return <Descriptionlist data={caption}></Descriptionlist>;
    }}
  </Story>
</Canvas>

### Behaviors

#### States

The text sizes of Lists for Desktops, Laptops and Tablets remains the same, while the text size for mobile is small for the convenience of the users as mobile screens are smaller in size.
